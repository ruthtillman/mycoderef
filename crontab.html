Using PuTTy to edit cron job/crontab (like VI)

Steps in PuTTY to access and edit

Step 1. Login to the server.

Step 2. Type crontab -e to open crontab. Scroll down (may need to use arrow keys) to view current commands list.

Step 3. Press Insert to make cron jobs editable.

Step 4. Edit file, then press Esc to stop editing.

Step 5. Type :wq to exit the crontab. You'll need to do this to get the message "installing new crontab"

Basic Cron Setup

Cron allows a great deal of specificity in how things are set up:

MIN — Minute 0-60

			HOUR — Hour [24-hour clock] 0-23

			MDAY — Day of Month 1-31

			MON — Month 1-12 OR jan,feb,mar,apr&ellip;

			DOW — Day Of Week 0-6 OR sun,mon,tue,wed,thu,fri,sat

			COMMAND — Command to be run Any valid command-line
MIN HOUR MDAY MON DOW COMMAND
*/10 * * * * wget http://mydomain.com/myphp_script.php

The command above will run every 10 minutes. The slash allows for repeating, The 

*
means it will be run every instance of the time period. Without the 
/
, it would run on the tenth minute of the hour. Without the asterisk, it would (I think) need an hour designated on which to run it.

Commas and dashes should be used to designate multiple items in any section. For example:

01,31 04,05 1-15 1,6 * /usr/bin/somedirectory/somecommand

would run on the 1st and 31st minutes of 4 and 5am on the Jan 1-15 and June 1-15.

30 0 * * sun,wed /usr/bin/somedirectory/somecommand

...would run at 12:30am (0030 hours) each Sunday and Wednesday.
